系統規格書

社團法人中華台北羽球3對3發展協會為推動全國性羽球3對3運動之組織化發展， 自行建置線上會員暨賽事管理平台，整合會員申請、賽事報名、繳費核對與後台治理， 取代現行人工紙本作業。本文件說明系統目的、架構、功能規格與資料治理方式， 供申請本會體育運動發展相關政府補助經費之技術佐證文件。

一專案背景與目的

本會為全國性人民團體，以推廣羽球3對3運動、辦理全國性錦標賽事、發展會員組織為宗旨。 隨著會員人數與賽事規模逐年成長，原以人工表單、Excel 與 LINE 群組彙整的會務作業方式， 已難以負荷報名資格審核、費用計算與繳費核對之工作量，亦缺乏可稽核之數位紀錄。

有鑑於此，本會自行規劃並開發「會員暨賽事管理資訊系統」，將入會申請、賽事報名、 費用試算、匯款核對與會籍管理等流程全面線上化，並建立具權限分級之後台管理機制， 使幹部得以即時掌握會務與財務狀況，降低人為疏漏，提升行政透明度與服務效率。 本系統已於本會官方網站正式上線運作，並將作為 2026 年 12 月 20 日「第一屆中華盃全國羽球3對3錦標賽」 之官方報名與會籍管理平台。

二系統目標與預期效益

建置本系統之核心目標，在於以最低之外部服務成本，達成會務與賽務作業之全面數位化：

  • 單一入口整合：官網、會員系統、賽事報名、繳費核對、後台治理共用同一資料庫，避免多平台資料不一致。
  • 自動化資格與費用計算：依會員資格、年齡、性別組合自動驗證報名資格並試算應繳金額，取代人工核對。
  • 可稽核之金流紀錄：所有繳費均以銀行轉帳＋末5碼比對方式留下數位紀錄，並可於後台匯出報表。
  • 組織治理透明化：後台儀表板即時呈現會員數、當日報名、待確認款項與累計收入，支援理監事會報告。

三使用對象與角色

系統依 Role 欄位劃分三種權限角色，並以伺服器端驗證確保跨角色資料存取安全：

Member

一般會員 / 選手

  • 以 Google／LINE／Facebook 第三方帳號登入
  • 申請入會、續會、編輯個人資料
  • 賽事報名、查詢報名與繳費紀錄
Staff

協會工作人員

  • 審核會員與報名繳費
  • 維護賽事、組別與公告內容
  • 匯出報名與財務相關報表
Admin

系統管理者（理監事 / 秘書處）

  • Staff 之全部權限
  • 使用者角色授權與帳號治理
  • 檢視整體儀表板與經營指標

四系統架構

本系統採用 Serverless 架構部署於 Vercel，前後端邏輯統一以 Next.js App Router 撰寫， 資料庫採用具自動備份能力之 Neon PostgreSQL，並透過 Auth.js 委託第三方平台完成身分驗證， 本會伺服器不儲存任何使用者密碼。

本節盤點系統現行之視覺識別與介面設計現況，並提出後續統一設計系統、優化介面流程之規劃方向， 作為使用者體驗精進與後續開發之依據。

（三）UI/UX 精進規劃

  • 統一設計系統文件：制定跨官網與後台之共用設計代幣（色彩、字級、間距、元件狀態），收斂現行雙視覺語彙之落差，並建立元件庫規範供後續開發依循。
  • 互動原型與可用性測試：以 Figma 建立正式互動原型，於重要功能（報名、繳費回填）上線前進行使用者測試，降低操作疑慮與客服負擔。
  • 無障礙檢核：依 WCAG 2.1 AA 標準檢視色彩對比、鍵盤可操作性與螢幕閱讀器相容性，提升不同族群會員之使用可及性。

六功能模組規格

（一）協會官網 Public Site

提供首頁、關於協會、成立宗旨、賽事資訊與聯絡我們等公開頁面，供一般民眾與潛在會員瀏覽，並內建線上聯絡表單。

（二）會員系統 Membership

  • 支援 Google／LINE／Facebook 第三方登入，免自建密碼儲存機制。
  • 入會申請採「銀行轉帳＋末5碼驗證」，避免串接第三方金流之額外手續費成本。
  • 身分證字號格式與檢查碼即時驗證，且設有一次性修改鎖定機制，防止重複冒用他人資料。
  • 會籍到期自動判斷，逾期提示續會；支援孤立會員資料（由報名流程建立）與使用者帳號之自動配對認領。

（三）賽事報名系統 Registration

  • 固定三人一隊，系統依出生日期即時計算年齡（不接受人工輸入年齡），並依組別驗證總年齡與個人最低年齡門檻。
  • 依組別自動限制性別組合（男子3P／女子3P／混合3P）與各組別名額，額滿自動關閉報名。
  • 依會員資格別（有效會員／新加入會員／非會員）自動計算應繳費用，並偵測同一選手跨組報名之「第2項」優惠試算。
  • 報名同時完成之健康與安全同意書電子簽署，留存數位紀錄。

（四）繳費與金流核對 Payment Verification

因應中小型體育團體之營運成本考量，系統不串接第三方金流服務，改採銀行轉帳＋匯款末5碼、轉帳日期回填之驗證機制： 會員完成轉帳後於系統登錄末5碼，工作人員於後台比對銀行明細後一鍵確認，系統自動更新會籍與報名之付款狀態，並同步啟用相應之會員資格。

（五）會員中心 Member Dashboard

提供數位會員卡（會員編號、到期日）、報名紀錄、付款紀錄與個人資料維護；系統會自動辨識入會費是否已併入賽事報名費用中，避免會員重複繳費之疑慮。

（六）後台管理系統 Admin Console

  • 經營儀表板：會員總數、有效會員數、當日報名數、待確認款項、累計實收金額。
  • 會員管理：搜尋、人工建檔、資料修正、繳費確認與續會處理。
  • 賽事與組別管理：建立賽事、設定組別年齡／性別／名額規則。
  • 報名管理：報名資料編輯、繳費確認、取消報名，並支援 Excel 匯出 供現場檢錄與財務對帳使用。
  • 公告管理：發布與下架協會公告。
  • 使用者權限管理：Admin 可調整帳號角色（Member／Staff／Admin）。

（七）照片與影片上傳中心 Media Upload

現行「協會照片」為固定於程式碼庫中之靜態圖片，需由工程人員手動更新，會員與工作人員均無法自行上傳。 照片與影片上傳中心將開放工作人員（後續評估開放會員）直接上傳賽事照片與精華影片， 經後台審核後自動發布於官網相簿與賽事花絮頁面，實作方式詳見第十二節「AWS 影音串流與檔案儲存建置規劃」。

七資料庫設計

資料庫採正規化設計，以 Prisma Schema 定義並透過型別檢查確保存取一致性，核心資料表如下：

八關鍵作業流程

（一）入會申請流程

1

第三方帳號登入

會員以 Google／LINE／Facebook 登入本系統。

2

填寫入會申請

填寫真實姓名、身分證字號、生日、聯絡方式，系統即時驗證格式與重複性。

3

匯款並回填末5碼

會員依系統提示轉帳年費，並登錄轉帳日期與帳號末5碼。

✓

後台審核啟用

工作人員核對銀行明細後於後台確認，系統自動啟用會籍並設定到期日。

（二）賽事報名與繳費流程

1

選擇賽事與組別

系統依報名起訖時間與名額狀態顯示可報名組別。

2

填寫隊伍與選手資料

三位選手資料、會員資格別即時驗證，系統同步試算全隊應繳金額。

3

簽署同意書並送出

完成健康與安全同意書電子簽署後送出報名，系統建立報名與待繳款紀錄。

4

匯款並回填資料

依系統顯示金額轉帳，並回填末5碼與轉帳日期。

✓

後台確認並啟用

工作人員確認款項到帳，系統同步更新報名狀態，並為「新加入會員」選手啟用會籍。

九資安與隱私保護機制

  • 免自建密碼儲存：全面採用 OAuth 2.0 第三方登入（Google／LINE／Facebook），本系統不儲存任何使用者密碼。
  • 角色權限控管：所有後台操作均於伺服器端驗證使用者角色（Member／Staff／Admin），未授權請求一律拒絕。
  • 個資最小化蒐集：僅蒐集辦理會籍與賽事報名所必要之姓名、身分證字號、生日、聯絡方式等資料。
  • 身分證字號防濫用機制：格式與檢查碼驗證、同賽事重複報名上限管制，並限制身分證字號僅能修改一次。
  • 伺服器端表單驗證：所有輸入均以 Zod 進行伺服器端二次驗證，避免前端繞過造成之資料異常。
  • 資料庫交易一致性：金流相關操作採資料庫交易處理，防止部分寫入失敗導致金額或會籍狀態錯誤。
  • 傳輸安全：全站透過 Vercel 平台以 HTTPS 加密傳輸，資料庫連線採用 Neon 加密連線。

十一永久網域與基礎建設永續規劃

本節說明現行網域與主機架構之潛在風險，並提出使本系統長期穩定存續、不受單一人員或第三方平台異動影響之建置規劃。

（一）現況與風險盤點

  • 網域現況：系統現行網址為 3v3.ek21.com，屬第三方平台之子網域，網域註冊與管理權責未完全歸屬本會法人名下，存在因平台政策變動或帳號異動而影響官網存續之風險。
  • 對外 IP 現況：現行主機服務（Vercel）採全球邊緣網路（Edge Network）架構，依請求來源就近路由，不提供單一固定對外 IP。此為現代 Serverless 平台之常態設計，有利於自動擴展與抵禦流量攻擊，但無法滿足部分機關或金融單位要求「來源 IP 白名單」之介接情境。

（二）建置規劃

1

網域法人化

以「社團法人中華台北羽球3對3發展協會」名義註冊正式頂層網域（建議 .org.tw 或 .tw），網域註冊人與付款帳戶均歸屬協會，避免因個人帳號或第三方平台異動而中斷服務。

2

DNS 雙路備援

導入雙 DNS 服務（如註冊商 DNS ＋ Cloudflare）互為備援，降低單一 DNS 服務中斷造成全站無法連線之風險。

3

固定 IP 節點（視需求）

僅針對確有固定來源 IP 需求之服務（如：影音串流入口、跨機關資料介接）於 AWS 建置具 Elastic IP 之獨立服務節點，與主網站（Vercel）分離部署，避免影響一般網頁瀏覽之高可用性。

✓

到期管理機制

建立網域與 SSL 憑證到期提醒機制，並將網域年費、AWS 服務費用編列於協會常態年度預算，避免因人員異動導致網域或服務中斷。

十二AWS影音串流與檔案儲存建置規劃

因應賽事規模擴大與「照片與影片上傳中心」功能需求，規劃導入 AWS 雲端服務作為影音與檔案之儲存、轉檔與傳遞骨幹， 取代現行將照片直接置於程式碼庫之作法。

（一）建置動機

  • 現行協會照片為靜態圖片，直接置於程式碼庫 public 目錄中，需仰賴工程人員手動更新版本並重新部署，工作人員與會員均無法自行上傳或管理。
  • 系統現有資料庫（Neon PostgreSQL）與主機（Vercel）均針對一般網頁資料設計，不適合直接儲存大型影片檔案或提供影音串流服務。
  • 隨賽事規模擴大，本會規劃提供賽事精華影片、選手花絮與直播內容，需具備可承載大流量之影音儲存與傳輸架構。
